1. Analisis Kebutuhan

A. Analisis Kebutuhan Fungsional
    Hal-hal yang dibutuhkan oleh sistem yang akan dibangun antara lain :
    1. Kemampuan untuk dapat melakukan order barang dengan menampilkan jumlah stok dan rincian barang yang tersedia.
    2. Kemampuan untuk menghitung stok barang dari proses keluar dan masuk barang.
    3. Kemampuan pengambilan keputusan dari order yang masuk untuk diterima atau ditolak.
    4. Cetak laporan barang keluar.
    5. Mengelola akun.
    B. Analisis Kebutuhan Non Fungsional
      1. Mengidentifikasi yang dibutuhkan oleh pengguna
      2. PC (Personal komputer) atau Laptop.
      3. Monitor, spesifikasi yaitu minimal layar 14inc.
      4. Processor Minimum Pentium Intel Celeron 2955U.
      5. Memori yang digunakan yaitu minimal 2 GB.
      7. Web Browser Google Chrome.
      8. Sistem Operasi Windows 7 atau diatasnya.
      9. Keyboard dan Mouse.
    C. Analisis Pengguna
      A. Admin Gudang
      1. Mengelola material barang.
      2. Mengelola data persediaan barang.
      3. Melihat daftar pesanan
      4. Melihat riwayat daftar pesanan.
      5. Melihat data stok barang
      6. Konfirmasi kirim barang
      7. Mengelola akun

      B. Manager
      1. Konfirmasi status pesanan (tolak/terima) .
      2. Melihat data persediaan barang.
      3. Melihat data order.
      4. Melihat riwayat daftar pesanan.
      5. Laporan pengeluaran barang.

      C. Client
      1. Memesan barang.
      2. Melihat data stok barang.
      3. Konfirmasi status penerimaan barang.

    D. Analisis Masukan Sistem dan Keluaran Sistem
      A. Masukan Sistem
      1. User terdiri dari admin, manager dan client.
      2. Data barang.
      3. Data stok.
      4. Data client.

      B. Keluaran Sistem
      1. Informasi stok barang..
      2. Informasi order.
      3. Informasi user.
      4. Laporan barang

    B. Entity Relationship Diagram (ERD)

  1. Tabel User

    2. Tabel user digunakan untuk menyimpan data pengguna agar bisa masuk ke aplikasi sistem informasi inventori dengan masing-masing level yang dibutuhkan yang berisi id, username, password, client dan level (user) . Berikut penjelasan isi tabel user :
    Nama Tabel : tbl_user
    Primary Key : Id_user
    Foreign Key : -

    No. Field Type Size Keterangan
    1 Id_user Interger 11 Primary key
    2 username Varchar 20 Username
    3 password Varchar 50 Password
    4 client Varchar 10 Client
    5 level Varchar enum admin,manager,client

  2. Tabel Client

    3. Tabel client digunakan untuk menyimpan data client pemesan yang berisi id, kodeclient, namaclient, kota, no_hp , alamat. Berikut penjelasan isi tabel client :
    Nama Tabel : tbl_client
    Primary Key : kodeclient
    Foreign Key : -

    No. Field Type Size Keterangan
    1 kodeclient Varchar 10 Primary key
    2 namaclient Varchar 30 Nama Client
    3 kota Varchar 20 kota
    4 no_hp Varchar 12 Nomor Hp
    5 alamat Varchar 100 Alamat

  3. Tabel Tambah Stok

    4. Tabel tambah stok digunakan untuk menyimpan data penambahan barang yang dimasukkan ke dalam gudang, Berikut penjelasan isi tabel tambah stok :
    Nama Tabel : tbl_tambahstok
    Primary Key : Id_tambahstok

    No. Field Type Size Keterangan
    1 id_tambahstok Interger 11 Primary key
    2 tgl date - Tanggal barang masuk
    3 namabarang Varchar 10 Kode Barang
    4 jumlah Integer 11 Jumlah
